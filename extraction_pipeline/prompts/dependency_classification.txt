You are given a list of ACE (Atomized, Causal, Explicit) sentences extracted from a student's comment about a news article. Your task is to build a **dependency graph** over these sentences.

## Task

For each sentence, determine which other sentences it **depends on**. A sentence S_b depends on sentence S_a if understanding or justifying S_b requires the information in S_a. Dependency can capture:
- **Causal links**: S_a describes a cause and S_b describes its effect.
- **Elaboration / evidence**: S_b elaborates on, explains, or provides evidence for S_a.
- **Conditional reasoning**: S_b states a consequence that is conditional on the claim in S_a.
- **Temporal / narrative continuation**: S_b continues a story or timeline begun in S_a.
- **Reference / anaphora**: S_b refers to an entity, event, or situation introduced in S_a (e.g., "this situation", "that time").

A sentence may depend on **zero, one, or many** other sentences. Sentences can depend on sentences that appear **before or after** them in the list — ordering does not restrict dependency direction.

## Constraints

1. The graph **must be a DAG** (Directed Acyclic Graph): there must be no cycles. If A depends on B, then B must NOT (directly or transitively) depend on A.
2. Every sentence from the input must appear exactly once in the output.
3. Each node's `id` must be a 0-based integer matching the sentence's position in the input list.
4. `depends_on` is an array of integer `id` values (may be empty).

## Output format

Return a single JSON object with exactly this shape:

```json
{
  "article_id": <article_id from input>,
  "comment_index": <comment_index from input>,
  "dependency_graph": [
    {
      "id": <int>,
      "sentence": "<the sentence text>",
      "depends_on": [<int>, ...]
    },
    ...
  ]
}
```
